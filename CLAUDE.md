# Fighters Legacy ŌĆö Claude Code Instructions

## Project Overview

GPL v3 general-purpose combat flight sim engine, inspired by Jane's Fighters Anthology (1998).
Cross-platform: Windows 10/11, Linux, macOS. Phase 2 (Modern-Particles Engine) is active.

The engine is fully content-agnostic. It knows nothing about FA or any specific game.
FA support lives in jomkz/fa-content. No FA-specific code belongs in this repo.

`ft-gui` is an old name for `fighters-codex` (a separate project). It has no role here. Phase 4 UI work follows the `IGui` HAL pattern (pure-virtual interface + swappable backends, same as `IRenderer`); no ft-gui references belong in this repo.

### Math library

**GLM** is the shared vector/matrix/quaternion library, linked as an INTERFACE dependency on `platform-hal`. Anything that links `platform-hal` (engine, game, tests) gets GLM automatically. Use `glm::vec3`, `glm::mat4`, `glm::quat`, etc.

### Renderer architecture (Phase 2)

`VkRenderer` (platform/vulkan/) uses Vulkan 1.3 dynamic rendering (`VK_KHR_dynamic_rendering`) ŌĆö no `VkRenderPass` or `VkFramebuffer` objects. Seven steps per frame (one compute dispatch + six rendering scopes):

1. **Shadow** ŌĆö up to `kNumCascades=4` PSSM cascades rendered into a depth array (`VK_FORMAT_D32_SFLOAT`, **forward-Z**, depth clear = 1.0); resolution and cascade count are runtime-configurable via `ShadowQuality` (`kShadowRes=2048`, 4 cascades = `ShadowQuality::High` default). Cascade matrices computed via tight bounding-sphere fit; PCF comparison sampler (`VK_COMPARE_OP_LESS_OR_EQUAL`). `ShadowUBO` bound at set 0, binding 2; `sampler2DArrayShadow` at set 0, binding 3. `ShadowQuality::Off` ŌåÆ `numCascades=0`, 1├Ś1 image allocated for descriptor validity; shader returns 1.0 immediately.
2. **Particle compute** ŌĆö `particle_sim.comp` dispatched before forward pass; advances age/pos for up to `m_maxParticles` slots (runtime-configurable via `ParticleDensity`; `kMaxParticles=8192` = `ParticleDensity::High` default; local_size_x=64). New particles written to a host-visible spawn staging buffer by CPU each frame then `vkCmdCopyBuffer`'d into the device-local pool SSBO (ring-buffer overwrite). Push constant `{dt, count, gravity, _pad}` (16 bytes). Barrier: COMPUTE_WRITE ŌåÆ VERTEX_SHADER_READ before forward pass.
3. **Forward (opaque)** ŌĆö Cook-Torrance PBR (GGX NDF, Smith geometry, Schlick Fresnel) with normal maps + ORM textures (set 1: base color / normal / ORM at bindings 0ŌĆō2). Geometry into HDR offscreen (`VK_FORMAT_R16G16B16A16_SFLOAT`) with **reverse-Z** depth (`VK_FORMAT_D32_SFLOAT`, far = 0.0, depth clear = 0.0, compare = GREATER). Depth storeOp = STORE (sky + particle + transparent + bloom passes read depth). **Winding/cull:** `cullMode=BACK`, `frontFace=VK_FRONT_FACE_COUNTER_CLOCKWISE` (shadow + transparent pipelines match). Combined with the projection Y-flip (`proj[1][1]=-f`), this renders **standard glTF CCW-from-outside** meshes correctly ŌĆö the winding cross-product must AGREE with the outward stored normal (what Blender exports). All authored meshes (`buildTerrainMeshGlb`, the builtin wedge/floor) follow this; `validate-mesh` flags inside-out meshes. (A prior `frontFace=CW` double-flipped with the Y-flip and rendered standard glTF inside-out ŌĆö masked by the old symmetric single-colour placeholder.)
4. **Sky + particles** ŌĆö combined rendering scope: sky fullscreen triangle (GREATER_OR_EQUAL, depth write off) followed by particle billboard draw (GREATER, depth write off, depth test on). Two instanced draws (`vkCmdDraw(6, m_maxParticles, 0, 0)`): additive pipeline (fire/explosion) then alpha pipeline (smoke). `particle.vert` reads particle SSBO and camera UBO from set 0 (bindings 0ŌĆō1); push constant `uint32_t renderAdditive` selects which blend-mode particles to emit (others output degenerate off-screen positions).
5. **Transparent** ŌĆö alpha-blended items (materials with `MaterialDesc::alphaBlend=true`) drawn back-to-front using `m_forwardAlphaPipeline` (depth write OFF, cull NONE, SRC_ALPHA/ONE_MINUS_SRC_ALPHA blend). Sorted in `recordCommandBuffer` by descending squared camera distance.
6. **Bloom** ŌĆö enabled when `RendererSettings::bloom=true`. Three half-resolution passes into `m_bloomImage`/`m_bloomAuxImage`: bright-pass threshold (`bloom_threshold.frag`), horizontal Gaussian blur, vertical Gaussian blur. `TonemapPush::bloomStrength` controls additive blend weight in the tonemap shader.
7. **Tonemap** ŌĆö Khronos PBR Neutral + optional bloom composite (binding 1 = bloom buffer) + optional FXAA (5-tap luma edge-detect + 3-sample blur). `TonemapPush` (16 bytes): texelSizeX/Y, enableFxaa, bloomStrength. Fullscreen HDR ŌåÆ swapchain (`B8G8R8A8_SRGB`).

**Note:** shadow passes use forward-Z (near=0, far=1); scene depth uses reverse-Z. These are independent depth spaces.

World convention: right-handed, Y-up, meters (matches glTF). Vulkan clip-space Y-flip handled in the projection matrix. **World positions are `double`/`glm::dvec3`** throughout the engine (`EntityTransform::pos`, `MsgEntityEntry::pos`, `EntityRenderEntry::position`, `CameraView::worldOrigin`). Camera-relative rendering subtracts `dvec3 worldOrigin` before GPU upload, casting the small relative offset to `vec3` ŌĆö float32-safe at any scale including planet-scale.

**Camera-relative rebase invariant (do not double-subtract):** the rebase happens exactly ONCE, on the CPU in double precision. `SceneRenderer` and `TerrainStreamer::getRenderItems` compute `relPos = worldPos ŌłÆ camera.worldOrigin` (dvec3) and bake it into the `RenderItem`/model transform; the shaders then treat `push.model` as already camera-relative and must NOT subtract `worldOrigin` again (`mesh.vert` does `gl_Position = proj┬Ęview┬Ę(model┬Ępos)` with no further offset). Shadow cascades are likewise built in camera-relative space (`computeCascades` does not add `worldOrigin`; `mesh.frag` samples shadows with the camera-relative position directly). A second subtraction in the vertex shader produced `worldPos ŌłÆ 2┬ĘworldOrigin`, pushing the whole scene `worldOrigin.y` metres below the camera (Ōēł575 m at the sandbox spawn) ŌĆö the "camera floats above the ground / entity tiny and unreachable" class of bug.

**Texture upload:** KTX2 Basis Universal ŌåÆ BC7 (desktop, if `VK_FORMAT_BC7_UNORM_BLOCK` supported) ŌåÆ ASTC 4├Ś4 (Apple Silicon, if BC7 absent) ŌåÆ RGBA32 fallback. All mip levels uploaded via `createGpuImageCompressed` using `ktxTexture_GetImageOffset` per mip. sRGB/UNORM views chosen per texture semantic (base color = sRGB, normal/ORM = UNORM). Normal maps use tangent-space flat normal default `{128,128,255}`; ORM defaults to all-ones linear white.

Runtime shader discovery: `VkRenderer::resolveShaderDir()` tries `SDL_GetBasePath()` + `"shaders/"` first, then macOS `.app` bundle path, then the build-tree `FL_SHADER_DIR` fallback. Release packages must stage `*.spv` into `dist/shaders/` (see `release.yml`).

**Renderer instantiation:** game and tool code must use `createVulkanRenderer()` from `platform/vulkan/VkRendererFactory.h` ŌĆö never include `VkRenderer.h` directly. `VkRenderer.h` pulls in `VkResources.h` ŌåÆ `vk_mem_alloc.h`, which is only on the private include path of `platform-vulkan`.

**GPU UBO / push constant sizes (after #39):** `CameraUBO` = 144 bytes (set 0 binding 0); `LightUBO` = 64 bytes (set 0 binding 1; +`fogParams` vec4 for density/startDist/timeOfDay); `ShadowUBO` = **288 bytes** (set 0 binding 2; `numCascades uint32_t` + `_pad[3]` appended after `splitDepths` in #235). `SkyPushConstants` = **128 bytes exactly** ŌĆö fills the Vulkan-guaranteed minimum push constant budget. Do NOT add more fields to `SkyPushConstants` without first migrating the sky pass to use a UBO. `ForwardPushConstants` = 96 bytes (model + baseColorFactor + metallic + roughness + `shadingMode float`: 0 = normal PBR, 1 = terrain elevation/slope tint, 2 = debug per-face colour; driven by `kRenderFlagTerrain`/`kRenderFlagDebugFaceColor` on the `RenderItem`). Sky pass uses FBM procedural clouds driven by `skyParams.w` (cloudCoverage) and `fogParams` (density/startDist/timeOfDay).

**GLM extension headers:** `VkRenderer.cpp` requires `<glm/gtc/matrix_transform.hpp>` (for `glm::lookAt`) and `<glm/ext/matrix_clip_space.hpp>` (for `glm::orthoZO`). `engine/render/RenderSnapshot.h` and `engine/entity/EntityManager.cpp` require `<glm/gtc/quaternion.hpp>` (for `glm::quat`). These are not in `<glm/glm.hpp>` core ŌĆö always include them explicitly.

### Server-client architecture (Phase 2, feat/sandbox-server-client)

**Single-player**: the game client spawns the `fl-server` binary as a subprocess bound to `127.0.0.1:4778` (`LocalServer` in `game/fighters-legacy/`), then connects to it as a normal ENet client ŌĆö no server infrastructure code lives in the game binary. `fl-server` is the identical dedicated server used for multiplayer. `LocalServer::start()` generates a 24-char hex session token, then spawns fl-server with `[port] 1 --bind <addr> --admin-token <token>` and waits for the "listening on" log line before returning a `StartResult` (`Ok`, `SpawnFailed`, `BindFailed`, or `Timeout`); the server thread stores a static failure string in `Game::Data::serverFailMsg` (`std::atomic<const char*>`) which `LoadingScreen` polls via a `getStartFailMsg` callback to show a specific message immediately.

**Game-layer classes** (in `game/fighters-legacy/`):
- `LocalServer` ŌĆö subprocess launcher; wraps `platform/Subprocess`; finds fl-server via `FL_SERVER_DEV_PATH` (CMake-injected dev-build path) or `SDL_GetBasePath()` (release co-located). `start()` returns `StartResult` (`Ok`, `SpawnFailed`, `BindFailed`, `Timeout`), generates a 24-char hex session token, and passes it via `--admin-token`; `sessionToken()` returns it for use by `makeNetworkAdminSender` in `main.cpp`. `registerConsoleCommands(registry, serverCommand, ...)` wires the pre-built `serverCommand` function (a `MsgAdminCommand` sender) into `CommandContext`.
- `ClientNetEventHandler` (`ClientNetEventHandler.h`) ŌĆö `INetworkEventHandler` for the game client; parses all message types and calls `bridge.publishExternal()` + `tickAlpha.markNewTick()` on each `WorldSnapshot`. Holds `ClientTickAlpha tickAlpha` for render interpolation (see below).
- `CameraInput` ŌĆö owns the 13 previously loose camera orbit state variables; translates SDL keyboard/mouse input to `CameraController` updates.

**Four independent overlay layers** submitted each frame via `IRenderer`:
1. **Cockpit HUD** (`fl::IHud*` ŌåÆ `FlightHud` builtin): `submitOverlayElements(activeHud->elements())` ŌĆö Cockpit mode only. Per-aircraft; `activeHud` swapped to content-pack implementation when provided (future phase).
2. **Windshield rain/snow** (`fl::WindshieldRain`): `submitOverlayElements(windshieldRain.elements())` ŌĆö Cockpit mode only.
3. **Server notices** (`ServerNotice`): `submitOverlayElements(serverNotice.buildElements())` ŌĆö any camera mode; set by `ClientNetEventHandler` on `MsgServerNotice`.
4. **Game console** (`GameConsole`): `setConsoleElements(gameConsole.elements())` ŌĆö any game state including future main menu.

**Render alpha**: previously computed by `serverLoop.shellTick()` (in-process sim thread). Now computed client-side by `ClientTickAlpha::get()` ŌĆö a wall-clock timer reset on each received `MsgWorldSnapshot`. No server infrastructure needed.

**Console commands** (`spawn`/`kill`/`tp`/`set_weather`): `CommandContext::serverCommand` (a `std::function<void(std::string_view)>`) sends a `MsgAdminCommand` (0x06) over ENet using the per-session token (`makeNetworkAdminSender` in `game/fighters-legacy/main.cpp`). In single-player the token is generated by `LocalServer::start()` and passed to fl-server via `--admin-token`. `nullptr` in multiplayer until the connection UI exists (future issue). The server admin console also has a `tp <idx> <x> <y> <z>` command.

**Platform layer**: `platform/Subprocess` (`Subprocess.h/.cpp`) provides cross-platform child process management (`CreateProcess`/`posix_spawn`, stdin/stdout pipes, liveness check, graceful+forceful stop). All `#ifdef WIN32` guards are confined to `Subprocess.cpp`; callers are `#ifdef`-free. `platform/StdoutLogger.h` is a shared header-only `ILogger` implementation for headless binaries.

**Shared engine utilities** (added in this refactor): `engine/config/ConfigFile.h` (`fl::ensureAndReadConfig`, `fl::writeConfigFile` ŌĆö raw-fstream config file helpers used by fl-server); `engine/net/NetworkUtils.h` (`fl::normalizeIp` ŌĆö IP normalisation previously duplicated in WorldBroadcaster.cpp and fl-server).

Private `bool m_connected{false}`; `char m_connectRefusalReason[64]{}` (buffer for refusal reason ŌĆö `const char*` stored in atomic points here, outlives `LoadingScreen` polling). Sends `MsgClientInput` each frame (arrow keys + shift + space).
- **ENet threading rule**: server `ENetHost*` is owned exclusively by fl-server's sim thread (via `WorldBroadcaster::onTick ŌåÆ net->service(0)`); client `ENetHost*` by the game's main thread. Never cross-thread.
- **`GameLoop::enqueueSimCallback(fn)`**: thread-safe lambda queue drained at the top of each sim tick before `ISimUpdate::onTick()`. Used by fl-server admin commands to mutate `EntityManager` safely. Console commands in the game client forward to fl-server's admin console via `serverCommand` callback instead.
- **Single-player render alpha**: computed by `clientHandler.tickAlpha.get()` ŌĆö `ClientTickAlpha` is a wall-clock timer reset on each received `MsgWorldSnapshot`. Replaces the removed in-process `serverLoop.shellTick()`.

### SimŌåÆrender bridge (Phase 2, PR 4)

`engine/render/SimRenderBridge` is a **lock-free triple-buffer** that ships a per-tick entity snapshot from the sim thread to the render thread. Three `RenderSnapshot` slots rotate: one owned by the sim, one in the atomic spare, one held by the render thread. `publish()` moves the completed snapshot into the spare (release fence); `tryAdvance()` atomically swaps the render slot with the spare when a newer tick is available (acq_rel fence). All three slot indices are always a distinct permutation of {0,1,2}.

- `EntityRenderEntry`: entityIdx, entityGen, typeIndex, position (**glm::dvec3** ŌĆö double-precision world position), orientation (glm::quat, w-first constructor), velocity (glm::vec3 for sub-tick extrapolation), damageLevel (uint8_t), playerOwned, throttle (uint8_t), fuelPct (uint8_t), abEngaged (bool ŌĆö afterburner physically lit), engineFailFlags (uint8_t ŌĆö `kEngineFail*` bitmask). Five `constexpr uint8_t` constants in `namespace fl`: `kEngineFailGeneric=0x01`, `kEngineFailLeft=0x02`, `kEngineFailRight=0x04`, `kEngineCompStall=0x08`, `kEngineFlameout=0x10`.
- **`publishExternal(RenderSnapshot)`** ŌĆö main-thread-only variant for network-client mode (no concurrent sim thread). Used by `ClientNetEventHandler` after parsing a `WorldSnapshot` packet.
- In server-client mode, do NOT call `EntityManager::setRenderBridge()` ŌĆö `WorldBroadcaster` owns serialisation; the render bridge is populated from network packets instead.
- `engine-render` CMake library is **unconditional** (no Vulkan dep) ŌĆö builds in CI without a GPU. `engine-entity` privately links `engine-render`; `engine-render` privately links `engine-content`. Any binary/test that links `engine-entity` gets both resolved automatically.

### Scene renderer (Phase 2, PR 5)

`engine/render/SceneRenderer` converts the per-tick entity snapshot into a `FrameScene` and calls `IRenderer::setScene` each frame. All work is main-thread.

- **MeshNameResolver**: `std::function<bool(uint32_t typeIndex, std::string& mesh, std::string& dmg)>` ŌĆö injected at construction to avoid a circular CMake dep between `engine-render` and `engine-entity`. In `main.cpp` the lambda captures `EntityTypeRegistry&`.
- **EffectResolver**: `std::function<std::string(uint32_t typeIndex, uint8_t damageLevel)>` ŌĆö optional; injected via `setParticleSystem(ParticleSystem*, EffectResolver)`. Called for each entity with `damageLevel > 0`; returns the particle preset name (e.g. `"explosion"`) or empty string. In `main.cpp` reads `EntityDef::damage` through the registry.
- **Mesh/material cache**: `getOrUploadMesh` / `getOrUploadMaterial` call `IRenderer::createMesh` / `createMaterial` once per unique mesh name; subsequent frames use the cached handle.
- **Camera-relative rendering**: `glm::vec3 relPos = glm::vec3(dvec3(entry.position) - camera.worldOrigin)` ŌĆö subtracts two `dvec3` values and casts the small result to `vec3` for GPU upload. Safe at planet scale.
- **Velocity extrapolation**: `rendered_pos = entry.position + entry.velocity * (alpha * kTickDt)` where `alpha = clientHandler.tickAlpha.get()` (client-side wall-clock alpha) and `kTickDt = 1/60 s`.
- **Damage variant**: if `entry.damageLevel > 0` and `EntityDef::classicDamageMesh` is non-empty, the damage mesh is loaded instead; `kRenderFlagDamaged` is set on the `RenderItem`.
- **Sort**: opaque items sorted front-to-back by squared camera-relative distance to minimise overdraw.
- **Draw-distance cull**: `setDrawDistance(float km)` configures the entity cull radius; entities beyond it are skipped before RenderItem construction. Wired from `RendererSettings::drawDistanceKm` in `main.cpp`.
- **Builtin placeholder fallback**: when `MeshNameResolver` returns false, `meshName` is empty, or mesh upload fails, the entity is rendered with a builtin directional **wedge** (~10 m, +X forward; flat bottom on the ground, flat vertical back, single nose vertex forward ŌĆö origin at the ground-contact point). Builtin entity items get `kRenderFlagDebugFaceColor`, so the forward shader colours each face distinctly from its normal (bottom=red, back=green, right=blue, left=yellow) ŌĆö orientation is unambiguous. `setBuiltinFloor(true)` appends a 4 km flat floor plane (off in the sandbox ŌĆö terrain replaces it). Geometry is in `engine/render/BuiltinGeometry.h`, generated by `tools/gen_builtin_glb.py` (standard CCW-from-outside winding; passes `validate-mesh` clean).
- `SceneRenderer::renderFrame(alpha, camera, env, extraEmitters={})` calls `tryAdvance()` internally; callers must NOT also call `renderBridge.tryAdvance()`. `extraEmitters` is merged with entity-damage effects when a ParticleSystem is set.

### GPU particle system (Phase 2, PR 6)

`engine/render/ParticleSystem` manages per-frame emitter emission on the CPU. `VkRenderer` handles GPU simulation and rendering.

- **ParticlePreset** (`engine/render/ParticleSystem.h`): spawnRate, particleLifetime, initialSpeed, colorStart/End, sizeStart/End, additive flag, emitDirection (glm::vec3, default {0,1,0} = upward hemisphere; {0,-1,0} for rain/snow), coneHalfAngleDeg (float, default 90.0 = full hemisphere; rain=20┬░, snow/storm_snow=80┬░).
- `ParticleSystem::registerPreset(name, preset)` ŌĆö register by name. `emit(name, worldPos, intensity=1)` fills a `ParticleEmitterState` from the preset and appends to the per-frame list. `reset()` + `emitters()` follow the per-frame accumulate-then-read pattern. `getPreset(name) const ŌåÆ optional<ParticlePreset>` ŌĆö returns the registered preset or nullopt (used by callers that need to build `ParticleEmitterState` manually for the `extraEmitters` path).
- **GPU pool**: `m_maxParticles` slots (runtime-configurable via `ParticleDensity`; Low=512, Medium=2048, High=8192 [default], Ultra=16384) in a device-local SSBO (`GpuParticle` ŌĆö 80 bytes: pos+age, vel+maxAge, colorStart, colorEnd, sizeStart/sizeEnd/additive/_pad). Ring-buffer spawn: `VkRenderer` maintains a per-emitter float accumulator (`m_spawnAccum`, a `std::vector<float>` indexed by emitter position in `particleEmitters`, reset to zero when the list count changes) to carry fractional remainder across frames ŌĆö `toSpawn = static_cast<uint32_t>(accum); accum -= float(toSpawn)` ŌĆö so presets with `spawnRate < 60/s` still emit correctly at 60 fps. CPU writes up to `m_maxSpawnPerFrame` new particles (ratio 1:16 of pool size) to a per-frame host-visible staging buffer, `vkCmdCopyBuffer` into the pool. Particle SSBO + staging buffers are recreated when `ParticleDensity` changes via `recreateParticleResources()`.
- **Compute**: `particle_sim.comp` (local_size_x=64) decrements age, advances pos by vel├Śdt, applies gravity. Dead particles (ageŌēż0) written as zero and skipped in vertex shader.
- **Render**: `particle.vert` reads SSBO (set 0 binding 0) + camera UBO (set 0 binding 1); emits a 6-vertex camera-facing quad per instance (2 triangles). `particle.frag` outputs circular soft-edge colour with quadratic alpha falloff. Push constant `uint32_t renderAdditive` selects which particles each draw pass processes.
- **DamageDef.visualEffect hook**: `DamagePenalty::visualEffect` (string in `DamageDef`) is the preset name emitted by `SceneRenderer` via `EffectResolver` when an entity is damaged.

`engine/render/CameraController` is a **thin pose holder**: `setMode(mode)` + `setPose(eye, forward, up)` + `view(aspect)` ŌåÆ `CameraView`. It does NO per-mode math ŌĆö there is a single underlying free-fly camera and the modes are just constrained ways of driving `setPose()`, all computed in `game/fighters-legacy/CameraInput`. Switched via F1/F2/F4 in `main.cpp`:

- **Free mode** (F4, default): the base camera. WASD/QE move the eye anywhere, mouse (LMB drag) turns the view; W/S/A/D are horizontal, Q/E vertical. Movement is **frame-rate independent** (metres/second ├Ś frame dt, default 30 m/s, +/- adjusts 2ŌĆō1000), and the eye is clamped above the terrain (`TerrainStreamer::heightAt`) so it descends to the ground but not through it.
- **Chase mode** (F2): pose computed to sit behind the entity's tail and follow its heading; aims at the entity origin (the ground-contact point). The user cannot rotate it.
- **Cockpit mode** (F1): pose locked to the entity, eye raised to the body centre, looking along the entity forward + RMB look offset. The ownship is hidden via `SceneRenderer::setHiddenEntity()` (shadow-only, `kRenderFlagShadowOnly`) ŌĆö not by back-face culling.
- **Projection**: infinite reverse-Z perspective hand-built from `f = 1/tan(fovY/2)` (default fovY = 60┬░): `proj[0][0]=f/aspect`, `proj[1][1]=-f` (Vulkan Y-flip), `proj[2][3]=-1`, `proj[3][2]=near`. `VkRenderer` reads `proj[3][2]` as the near-plane value for shadow cascade split.
- **`view()`** sets `cv.worldOrigin = m_eye` and builds `cv.view = glm::lookAt(origin, forward, up)` with the camera at the relative origin (guards against forwardŌłźup). The geometry is rebased against `worldOrigin` on the CPU (in double precision) before upload ŌĆö see the camera-relative invariant in the renderer section.
- **`CameraView::worldOrigin`** is `glm::dvec3`. `VkRenderer` casts it to `vec3` only at GPU upload sites (shadow cascade center, particle origin).

### World terrain and theaters (feat/world-terrain)

The engine uses a single continuous **world terrain** rather than per-theater heightmap grids. Theaters are geographic regions (bounding boxes in world coordinates) within this single terrain, not separate maps. Players can fly anywhere in the world in one session; theater boundaries are mission conditions, not engine limits.

**Terrain ID:** `"world"` is the canonical ID. fl-base-pack provides global coverage; theater content packs override individual chunks at higher mod priority via `IContentPack::resolveTerrainChunk()`.

**Chunk format:** 16-bit grayscale PNG, 513├Ś513 pixels, **15,360 m per chunk** (512 intervals ├Ś 30 m = native Copernicus GLO-30 resolution ŌĆö no upsampling needed). Three LOD levels: LOD 0 = 513├Ś513, LOD 1 = 257├Ś257, LOD 2 = 129├Ś129. Chunk path convention: `terrain/<id>/lod<n>/chunk_<x>_<y>.png` (all lowercase, 4-digit zero-padded coordinates).

**LOD ring distances** (for a 15,360 m chunk): LOD 0 within 3├Ś3 chunks (~46 km), LOD 1 within 5├Ś5 (~77 km), LOD 2 within 7├Ś7 (~107 km); evict beyond ~120 km. Maximum ~83 chunks in memory at steady state.

**`IContentPack::resolveTerrainChunk(terrainId, chunkX, chunkY, lod) ŌåÆ optional<string>`**: pure virtual ŌĆö implemented in #172. `FolderContentPack` probes `terrain/<id>/lod<n>/chunk_<x>_<y>.png` under its mod dir (zero-padded 4-digit coords). `AssetManager::resolveTerrainChunk()` walks the priority stack; first-wins, not cached. `TerrainStreamer` calls this to resolve the path before queuing `IAsyncFilesystem::readFileAsync()`. `MockContentPack` in `test_content_system.cpp` has a configurable `chunkPaths` map (key format: `"terrainId:x:y:lod"`) for priority-stack tests; all other test mocks stub it as `return std::nullopt;`. Test `IContentPack` mocks derive from a shared `NullContentPack` (`tests/mock_content.h`) and override only what they exercise (`MockContentPack` mesh/asset maps, `MockTerrainPack::resolveTerrainChunk`, the music fixtures' `loadAudio`, etc.), so a new `IContentPack` pure virtual only needs a default added to that one header ŌĆö the old 'update all 6 mocks' tax is gone. `BorrowedPack` in `test_content_system.cpp` is a deliberate non-owning forwarding proxy (delegates every method to a test-owned pack), so it still tracks the interface.

**`TerrainStreamer`** (`engine/render/TerrainStreamer.h`): manages chunk lifecycle. Constructor: `TerrainStreamer(TerrainManifest, AssetManager&, IAsyncFilesystem&, IRenderer*)` ŌĆö `IRenderer*` nullable for headless. Inherits `IAsyncFilesystemHandler`; registers/deregisters itself as the sole async FS handler. `update(dvec3 cameraPos)` computes desired chunk set (LOD rings), evicts out-of-range chunks, queues async reads or generates procedural fallback (rate-limited to `kMaxProceduralPerUpdate=8` chunks/frame to avoid first-frame hitch). `getRenderItems(dvec3 worldOrigin) ŌåÆ vector<RenderItem>` returns best-available LOD per (cx,cy) for the 7├Ś7 ring. `heightAt(double x, double z) ŌåÆ double` bilinear interpolation from the LOD0 chunk only; returns 0.0 if not loaded. `surfaceAt()` stub returns 0. `chunkCount() ŌåÆ size_t` exposed for tests. LOD ring semantics: each LOD maintained for its full concentric square ŌĆö LOD 0: 3├Ś3 = 9 entries, LOD 1: 5├Ś5 = 25 entries, LOD 2: 7├Ś7 = 49 entries; **83 total** (9+25+49), all independent, enabling seamless LOD transitions. Mesh generation uses `buildTerrainMeshGlb()` (see `TerrainMeshBuilder.h`). Single shared `m_terrainMat` for all chunks. Wired in game `main.cpp` via `sceneRenderer.setTerrainStreamer(&ts)` and `p.asyncFilesystem->service()` + `ts.update()` each frame. In `fl-server`, wired headlessly: `asyncFilesystem->service()` + `terrainStreamer.update()` called from the main admin loop (~20 Hz); `m_renderer = nullptr` so GPU upload is skipped and only heightmap data is kept. PNG decode (for content-pack chunks) via `decodeTerrainChunkPng()` in `TerrainChunkIO.cpp` ŌĆö `STB_IMAGE_STATIC` already defined there to prevent ODR conflict with `VkResources.cpp`.

**Terrain rendering:** chunks are submitted as standard `RenderItem`s via `IRenderer::createMesh()`/`createMaterial()` ŌĆö no new `IRenderer` pure virtuals, no `FrameScene` changes, no `MockRenderer` updates required for Phase 2.

**fl-server** initializes `SDL3Filesystem` + `SDL3AsyncFilesystem` + `ModLoader` + `AssetManager` + headless `TerrainStreamer` (null renderer, #174) to serve `heightAt()` for authoritative physics. CMake deps added: `engine-render engine-content platform-sdl3`. Content packs are loaded via `ModLoader` from the `mods/` subdirectory of the server's working directory; the builtin procedural terrain (FBM, ~550 m base) is used when no content packs are present. fl-server pre-spawns no entities; one `"builtin:debug-entity"` is spawned per connecting peer (via `WorldBroadcaster::onConnect`) at `heightAt(x, z) + AGL`. Peer spawn positions are pre-cached via `terrainStreamer.heightAt(x, z) + aglOffset` (from `[spawn]` config) before `gameLoop.start()` and stored in `m_spawnPoints`; `setGroundElevation` seeds only the physics floor used by `FlightIntegrator::step`. `TerrainStreamer::heightAt()` remains main-thread-only.

**DEM source:** Copernicus GLO-30 (ESA, 30 m global, free). `tools/gen_terrain_chunks.py` (#176) converts GeoTIFF ŌåÆ chunk PNGs at all 3 LOD levels. fl-base-pack `scripts/build_terrain.sh` downloads tiles from AWS Open Data (`s3://copernicus-dem-30m/`) and invokes the tool.

**Wire protocol:** fully documented in `docs/network-protocol.md`. `kProtocolVersion = 1`; `MsgHello` is 4 bytes (first reliable packet on connect; client disconnects on version mismatch); `MsgEntityEntry::pos[3]` is `double` (72 bytes, naturally aligned, `pos` @0; `abEngaged` uint8_t at offset 68, `engineFailFlags` uint8_t at offset 69 added in #286); `MsgConnectAck` is 16 bytes (includes `assignedEntityIdx/Gen`; `planetRadiusKm float32` @12 ŌĆö planet sphere radius in km, Earth default = 6371); `MsgClientInput` is 48 bytes (clientŌåÆserver, unreliable channel 1; `seqNum uint32_t` @4 guards against out-of-order/duplicate delivery; `tickIndex uint64_t` @8 enables per-peer one-way delay estimation); `MsgWeatherState` is 20 bytes (`MsgId::WeatherState=0x04`, unreliable ~6 Hz, additive ID); `MsgServerNotice` is 64 bytes (`MsgId::ServerNotice=0x05`, reliable, serverŌåÆclient shutdown countdown; additive ID); `MsgAdminCommand` is 128 bytes (`MsgId::AdminCommand=0x06`, clientŌåÆserver reliable; `reqId uint16_t` @2, `token char[30]` @4, `command char[94]` @34; additive ID); `MsgAdminResponse` is 128 bytes (`MsgId::AdminResponse=0x07`, serverŌåÆclient reliable; `reqId uint16_t` @2, `text char[124]` @4 [fast-path, results Ōēż 123 chars]; additive ID); `MsgAdminResponseChunk` is 512 bytes (`MsgId::AdminResponseChunk=0x0A`, serverŌåÆclient reliable; `flags uint8_t` @1 [bit 0=`kChunkFlagEnd`], `reqId uint16_t` @2, `seqNum uint16_t` @4, `body char[506]` @6 [505 usable]; streaming path for results > 123 chars; additive ID); `MsgMotd` is variable-length (`MsgId::Motd=0x08`, reliable, serverŌåÆclient once after `MsgConnectAck` when `[server].motd` non-empty; `MsgMotdHeader` 4 bytes [msgId @0, reserved @1, displaySeconds uint16 LE @2] + null-terminated UTF-8 text @4 capped at `kMaxMotdBytes=65535`); `MsgConnectRefusal` is 64 bytes (`MsgId::ConnectRefusal=0x09`, reliable, serverŌåÆclient; `code` uint8_t at offset 1 (`ConnectRefusalCode`); `reason char[62]` at offset 2 [null-terminated UTF-8]; sent before `disconnectPeer()` on every `onConnect` rejection; additive ID, old clients discard); `MsgLanBeacon` is 74 bytes (`MsgId::LanBeacon=0x10`, raw UDP broadcast, not ENet ŌĆö received by `DiscoveryListener`). Parse via `engine/net/WireCodec.h` (`readMsg`/`viewMsg`). Structs are unpacked and naturally aligned (full normalization in the Phase 2ŌåÆ2b cleanup). #92 (version negotiation), #142 (authoritative game protocol), #170 (double-precision positions), #91 (LAN discovery), #39 (weather/time-of-day), #90 (shutdown countdown), #229 (network admin channel), #231 (MOTD delivery), #302 (server-controlled MOTD display duration), #343 (connect refusal reason), #239 (admin response streaming), and #361 (admin reqId correlation) are all merged.
